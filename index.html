Getting Started

All the necessary dependencies and tools to install

For the Backend part you have to install docker, node. Use the package manager npm to install all the important dependencies.

 npm install | npm i

Environment File

It is very important to create an .env file with the following environment variables.

MASTER=true
APP_ID=''
APP_ACCESS_KEY=''
FAKE_TTN_CONNECTION=false
DEBUG=true

The meaning of all these variables are written in the Dossier.

Start the Backend

To start and stop the backend these following commands can be used

docker-compose up
# starts all docker containers at the same time
docker-compose down 
# stops all the docker container at the same time

Tests

To start the tests you have to stop the running containers with this command. 

docker-compose -f docker-compose.yml -f docker-compose.test.yml down

After that you can start the test with: 

npm test

Endpoints and Channels

Here you can see all the Enpoints and Channels of the Backend

Endpoints: 

Authentication
   - login, params: ({email:string, password:string})        
   - logout, params: (undefined)                                       
   - changePassword, params ({currentPassword, newPassword})   
   - resetPassword, params ({email: string})                      
NodeClusters
   - getNodeClusters, params: (undefined) 
   - getNodeClusterById, params: (nodeClusterId: string)
   - createNodeCluster, params: (name: string) 
   - removeNodeClusterById, params: (id: ObjectId) 
   - addNodeToNodeCluster, params: ({nodeId: ObjectId, nodeClusterId: ObjectId}) 
   - removeNodeFromNodeCluster, params: ({nodeId: ObjectId nodeClusterId: ObjectId}) 
   - updateNodeClusterVisibility, params: ({public: boolean, nodeClusterId: ObjectId}) 
Nodes
   - requestNodeOwnership, params: ({devId:string, serialnumber: string})
   - updateNodeVisibility, params: ({public: boolean, nodeId: ObjectId})
   - removeNodeOwnership, params: ({devId:string, serialnumber: string})
Measurement
   - getLatestHighestMeasurement, params: ({nodeClusterId: ObjectId, expiredMeasurementsMinutes: number = 20})
   - getMeasurementIntervalResults, params: ({nodeClusterId: ObjectId, startTimestamp: number, endTimestamp: number, resultType: MeasurementIntervalResultType })
   - getMeasurementsOfNodeCluster, params: ({nodeClusterId: ObjectId, timeDeltaMin:number = 60})
AccountRequest
   - requestAccount, params: ({email: string, fullname: string, message: string})
   - getAccountRequests, params: (undefined)
   - replyToAccountRequest, params: ({accountRequestId: ObjectId, accept:boolean})

Channels: All of these have two channels, one for Owner and one for Visitor 

Measurement
   - MeasurementIntervalResults
    - socket.subscribe("MeasurementIntervalResults-${Owner | Visitor}-${NodeClusterId}-${Typeof MeasurementIntervalResultType (AVG | MIN | MAX)}")
   - LatestHighestMeasurement
    - socket.subscribe("LatestHighestMeasurement-${Owner | Visitor}-${NodeClusterId}")
   - GetMeasurementsOfNodeCluster
    - socket.subscribe("GetMeasurementsOfNodeCluster-${Owner | Visitor}-${NodeClusterId}")
